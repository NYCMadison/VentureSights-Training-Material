# 任务体系梳理：可对外输出的 Agent 产品化清单

**文档用途**：将内部「场景任务（STOSS）+ Skill 任务包 + 虚拟角色（Subagent）」体系，整理为**对外可售、任务边界清晰、契合科技投行与产业研究**的 Agent/SKU 清单，供商务、产品与路演使用。  
**版本**：v1.3  
**整理日期**：2026-05-20（v1.3：日更情报对外仅 **全产业机会研究链**；不含维辰 VentureDaily / 播客 / 海淀报送等内容线）  
**关联文档**：《科技投行体系_虚拟团队与功能全景》《科技投行相关_Skills介绍》《产品介绍_对外》《industry-analysis-agent_产品介绍_对外》《financial-advisory-agent_产品介绍_对外》

---

## 一、体系三层：对外怎么讲

| 层级 | 对内作用 | 对外 Agent 化用法 |
|------|----------|-------------------|
| **STOSS（场景任务）** | 规定「做什么、交什么、交付前自检什么」 | **一个 Agent 产品 = 一个或一组场景 + 固定交付清单** |
| **Subagent（虚拟角色）** | 规定「谁来做」（分析 / 表达 / 质检 / 财务等） | **不单独售卖角色**；写入 Agent 流水线说明即可 |
| **Skill（任务包）** | 可重复 SOP：步骤、触发、产出形态 | **最适合产品化**：可写进 SOW、可映射 API `task_type` |

**对外一句话**：Agent = **场景边界 + Skill 编排 + 知识库 + 质检门禁**；虚拟团队是内部工序，不必对客户枚举十几个角色名。

**标准流水线**（高 stakes 交付）：**分析 → 表达 → 质检**；含事实主张的对外材料验证深度**至少 L2**；事实与观点分离；对外只交付干净定稿。

## 二、已产品化的 API Agent（第一梯队）

| Agent | 覆盖任务 | 任务边界 | 科技投行 / 产业研究契合 |
|-------|----------|----------|-------------------------|
| **Industry Analysis Agent** | 产业深度、企业分析、区域产业、标的名单、PE 观察清单、VC 初判、热点传导简报等 | 对外产品介绍已列用例 1–7 | **高** — 咨询、PE/VC、并购、园区 |
| **Financial Advisory Agent** | Teaser/IM 行业章、亮点+风险、买方 Q&A、卖方行业附件、可比叙事等 | 典型问答样例已配套 | **高** — FA、ECM、卖方顾问 |

**共同底座**：知识库 + 预设产业框架与产业图谱 + 数据接口 + 有纪律的公开信息；**不提供 Web 控制台**，以 API 为触达方式。

## 三、五类场景任务（STOSS）→ 对外 Agent 映射

以下五类均有**完整任务拆解与交付清单**，默认含双路径知识库检索、免责与版权块；分析报告类**交付前必经数据质检**。（**危机管理**不纳入本对外清单。）

| 场景 | 约定交付物 | 建议对外 Agent 名 | 投行/研契合 | 备注 |
|------|------------|-------------------|-------------|------|
| **场景一：产业分析** | 完整产业报告（框架+数据+叙事+图表+免责） | 产业深度研究 Agent | ★★★★★ | 可并入 Industry Analysis Agent |
| **场景二：企业分析** | 企业分析文档（数据与叙事+图表+免责） | 企业分析 Agent | ★★★★☆ | 可并入 Industry Analysis Agent；**不设**基础信息尽调 SKU |
| **场景三：新闻获取与总结** | 日更情报（见第四节全产业机会研究链） | **产业日更 + 战略面板 Agent** | ★★★★☆ | 对外唯一日更 SKU |
| **场景四：VC 尽职调查** | 投资 memo / VC 尽调（含核心六问） | **VC 尽调 Agent** | ★★★★★ | **对外唯一尽调 SKU** |
| **场景五：区域产业分析** | 区域产业综述报告 | 区域产业 Agent | ★★★★☆ | 可接产业集群图谱 |

### VC 尽职调查（对外唯一尽调口径）

| 项 | 说明 |
|----|------|
| **触发（含其一）** | VC 尽调、投资 memo、一级市场分析、标的筛选、为 VC 做尽调、deal memo、investment memo |
| **约定交付** | 投资 memo / 尽调摘要 / 标的筛选清单（VC 框架维度 + **核心六问** + 数据与叙事 + 图表 + 免责） |
| **对外 Agent** | **VC 尽调 Agent**（场景四） |

**对外说明**：本清单**不单独提供**「基础信息尽职调查」模版或 SKU。客户泛称「尽调 / due diligence」且属 **VC / 一级市场** 语境时，一律按 **VC 尽职调查** 交付；并购控股权等深度 DD 材料见并购结构、技术尽调清单等 **交易支持** 能力，不替代持牌尽调结论。

### 用户定义分析框架（建议 API 参数 `framework_id`）

| 框架 | 典型触发（中英文） | Agent 化 |
|------|-------------------|----------|
| 产业链商业机会分析 | 商业机会、环节爆发顺序、industry chain commercial opportunity… | 是 |
| 技术路线对比 / 技术路径分析 | 技术路径、路线对比、technology roadmap analysis… | 是 |
| 政策影响分析 | 政策对产业/企业影响、policy impact analysis… | 是 |

命中时**必须先按框架步骤成稿**，再进入场景任务；是产业研究 Agent 相对通用 Chat 的**差异化能力**。

## 四、日更情报：全产业机会研究链（对外唯一口径）

| 项 | 说明 |
|----|------|
| **默认触发** | 「今日新闻 / 每日新闻 / 产业新闻总结」等**全产业**泛称（机构投研、战投日更场景） |
| **必选交付** | 十赛道 **机会假设包** + 各赛道 **产业叙事分析** + **十赛道合订** + 索引；定稿后默认后置 **十五五战略面板对外报告（V15）** |
| **对外 Agent / SKU** | **产业日更 + 战略面板 Agent**（对应场景三） |
| **主要客户** | 机构投研、战投、多赛道覆盖的买方研究与中台 |

**价值主张**：把当日要闻沉淀为**可检验的机会命题、机制链与战略面板**，而非标题堆叠或条式快讯。

**对外说明**：本清单**不纳入**维辰 VentureDaily 条式日报、播客脚本、海淀/政务报送包等**内容运营与政务报送线**（内部另线维护，不对外售卖为产业研究 Agent）。客户若仅需热榜或「今日侧重」信号钉，可作为本链前置输入，但**对外 SKU 以机会研究链 + V15 为准**。

## 五、Skill 组合包 → 逻辑 Agent（可写 SOW）

### 5.1 一级市场 / 尽调（优先独立）

| 逻辑 Agent | 主 Skill 链 | 明确产出 |
|------------|-------------|----------|
| **VC 尽调 Agent** | VC 尽调报告 + 商业尽调 + 技术尽调清单 + memo 财务章 + 工商/股东子链 | IC-Ready memo / 尽调主文 |
| **Coverage / 卖方研究 Agent** | 卖方 coverage memo + 产业框架 + 数据核验 | 可投资结论、催化剂、反驳点 |

### 5.2 并购与交易结构（FA 补强）

| 逻辑 Agent | 主 Skill 链 | 明确产出 |
|------------|-------------|----------|
| **并购结构 Agent** | 交易结构设计 + 跨境/并购税务清单 + 技术尽调清单 | 结构对比、税务/技术问题清单 |
| **ECM / 路演 Agent** | ECM 路演 playbook + 多时间尺度产业影响（可选） | 路演故事线、IR Q&A 骨架 |

可与 **Financial Advisory Agent** 合并为「交易材料套件」，或拆为子 `task_type`。

### 5.3 PE 二级

| 逻辑 Agent | 编排 | 明确产出 |
|------------|------|----------|
| **Scout / PE 二级 Agent** | 行业 PE 地图 → 持有期 / 业绩 / 催化剂 / 退出窗口 → 观察清单 | 带优先级或分级的 watchlist |

Industry Analysis Agent 用例已含 PE 观察清单；可单独品牌化 **Scout Agent**。

### 5.4 区域与数据产品

| 逻辑 Agent | 主 Skill | 明确产出 |
|------------|----------|----------|
| **区域产业集群 Agent** | 区域集群图谱 + 集群指标 Excel | 集群网络报告、七表工作簿 |

### 5.5 横切能力（嵌入各 Agent，不单卖）

- **数据核验**：分层验证 L0–L3，对外交付至少 L2  
- **知识库检索**：规则 + 语义双路径，分析报告类默认覆盖本地库 + 公开信息  
- **分析 / 成稿 / 统筹**：机制拆解、受众适配、多步编排（内部工序）  
- **企业事实**：工商、股东、竞品清单等子能力

## 八、产品化优先级矩阵

### 第一梯队（建议作为对外 SKU）

1. **Industry Analysis Agent**（已有）  
2. **Financial Advisory Agent**（已有）  
3. **VC 尽调 Agent**（场景四 + VC 尽调 skill；**对外唯一尽调 SKU**）  
4. **产业日更 + 战略面板 Agent**（全产业机会研究链 + V15）  
5. **Scout / PE 二级观察 Agent**（六支 PE skill 编排）  

### 第二梯队（行业 / 区域插件）

6. **区域产业 Agent**（场景五）  
7. **区域产业集群 Agent**（图谱 + Excel）  
8. **并购结构 + ECM 套件**（并入 FA 或独立「交易支持 Agent」）

## 九、落地建议（产品与 API）

1. **统一 `task_type` 枚举**：五场景 + 三注册框架 + VC 尽调 + PE Scout + 日更/V15 + 区域集群，与 API 文档一一对应。  
2. **尽调对外口径**：仅 **VC 尽职调查**；触发词与交付清单写入 VC 尽调 Agent 产品说明，不写「基础信息尽调」路由。  
3. **日更对外 SKU**：仅 **全产业机会研究链 + V15**；不写 VentureDaily / 播客 / 政务报送为对外 Agent。  
4. **每个 SKU 只承诺一种主交付物**（报告 / memo / 清单 / 报送包），质检深度写入 SLA（对外 ≥ L2）。  
5. **边界三句话**（与客户组织拼接）：  
   - 我方承担高密度信息工作与结构化论证；客户承担裁决、签字与对外责任。  
   - 法律、税务、审计等执业结论以人类专业人士为准。  
   - 建议从 1 个高频场景切入跑通编排与质检，再横向扩展。

## 十一、边界与合规（对外统一口径）

- Skills / Agent **不构成**：法律意见、税务法律意见、审计意见、投资建议或证券买卖建议。  
- 交付物默认定位为**研究底稿与背景附件**；投资决策、对外披露与法律责任由客户承担。  
- 对外正文**不暴露**内部核验过程、本地路径与调试留痕。  
- 无法核实的数据与事件须在正文中**显式标注局限**。
